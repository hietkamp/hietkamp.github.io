# Essence Way of Working

A static documentation site that presents a complete way of working built on the [OMG / SEMAT Essence standard](https://www.omg.org/spec/Essence/). The site is deployed to GitHub Pages via MkDocs and covers the Essence kernel plus four composable practices that together form a coherent, scale-independent method.

## What the site contains

| Page | Description |
|------|-------------|
| **Introduction** | The front door — what Essence is, the six element types, the kernel, and how the four practices compose into a logical flow that moves the alphas. |
| **The Essence Kernel** | Reference card deck for all seven Alphas (incl. the generalised System), fifteen Activity Spaces, and six Competencies across three areas of concern. |
| **Training** | A slide-style walkthrough of the kernel, the four practices, and the assessment workbook — how to use them together. |
| **Business Scenarios** | The TOGAF Business Scenario technique re-expressed as a composable Essence practice. Discovers and quantifies the need before anything is committed. |
| **Service Blueprint** | Visualises the end-to-end service from both the customer's and the provider's perspective — frontstage, backstage, and support processes. |
| **Minimum Viable Architecture** | Architect the minimum, just in time, to retire significant risks and guide delivery. Decisions over documents. |
| **Minimum Viable Governance** | Govern outcomes and guardrails, not gates. Decide at the last responsible moment, as close to the work as possible. |
| **Resources** | Downloads page — includes the EA Alpha Assessment workbook (Excel) and links to every page. |

The four practices stack on top of the kernel and are designed to run in order: Business Scenarios → Service Blueprint → Min. Viable Architecture → Min. Viable Governance. The loop nests at every level of scale.

## Repository structure

```
docs/                         # Source files (served as-is to GitHub Pages)
├── index.html                # Home / hub page
├── essence-introduction.html
├── essence-kernel.html
├── essence-training.html
├── business-scenarios-essence-practice.html
├── service-blueprint-essence-practice.html
├── mva-essence-practice.html
├── mvg-essence-practice.html
├── resources.html
├── EA_Alpha_Assessment.xlsx  # Downloadable assessment workbook
└── search-index.json         # Pre-built search index (generated by inject_search.py)
.github/workflows/deploy.yml  # GitHub Actions — builds and deploys on push to main
inject_search.py              # Utility: rebuilds the search index and updates all pages
mkdocs.yml                    # MkDocs config (used for gh-deploy only)
requirements.txt              # mkdocs + mkdocs-material
```

All pages are standalone HTML files that share an identical design system (Fraunces + IBM Plex fonts, warm paper background, CSS custom properties). Navigation, search, and accessibility fixes are injected into every page by `inject_search.py`.

## Local development

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
mkdocs serve          # preview at http://127.0.0.1:8000
```

## Deploying to GitHub Pages

Push to `main` — the GitHub Actions workflow runs `mkdocs gh-deploy` automatically and publishes the `docs/` content to the `gh-pages` branch.

To deploy manually:

```bash
source .venv/bin/activate
mkdocs gh-deploy
```

## Regenerating the search index

Run `inject_search.py` whenever page content changes. It rebuilds `docs/search-index.json` and re-injects the nav bar (with search) into all HTML files:

```bash
python3 inject_search.py
```

## Accessibility

All pages target WCAG 2.2 Level AA:

- Skip-to-main-content link on every page
- Full keyboard navigation for the Practices dropdown (Arrow / Escape / Space / Enter)
- `:focus-visible` outlines on all interactive elements (3 px, 9:1 contrast)
- Colour contrast corrected across the shared CSS custom properties
- Minimum 25 px touch/click target height on all nav elements
- Decorative elements marked `aria-hidden`
- Search combobox uses `aria-autocomplete`, `aria-expanded`, `aria-activedescendant`, and `role="listbox"` on the results list

## Content origin

The HTML pages originate from the Essence Way of Working project. They synthesise the OMG Essence standard with ideas from TOGAF, the TOGAF Business Scenarios method, service blueprinting (Shostack, 1984), and Use-Case 2.0. The composition is for learning purposes; keep your licensed standards as the authority.
